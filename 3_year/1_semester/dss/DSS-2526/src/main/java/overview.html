ğŸ¯ Design Patterns

  • Facade Pattern - CadeiaRestaurantesLNFacade, RestaurantesFacade, MenuPedidosFacade
    Simplifies access to complex subsystems with unified interfaces.
  • DAO Pattern - All *DAO classes
    Abstracts and encapsulates database access operations.
  • Singleton Pattern - DAO getInstance() methods
    Ensures single instance of data access objects.
  • MVC Pattern - View/Controller separation
    Separates presentation from business logic and data.
  • Subsystem Pattern - Modular organization
    Logically groups related functionality into cohesive subsystems.
